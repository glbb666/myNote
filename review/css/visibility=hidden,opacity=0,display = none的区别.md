### display = none,visibility=hidden,opacity=0的区别

> 渲染树的具体概念在浏览器中

| 特性                                            | `display: none`    | `visibility: hidden`           | `opacity: 0`         |
| ----------------------------------------------- | -------------------- | -------------------------------- | ---------------------- |
| **是否在渲染树中**                        | ❌ 不在              | ✅ 在                            | ✅ 在                  |
| **占据空间**                              | 不占据               | 占据                             | 占据                   |
| **是否触发事件**                          | 不触发               | 不触发                           | 触发                   |
| **是否可被子元素覆盖**                    | 不可（子元素也消失） | 可（子元素设 `visible`可显示） | 不可（视觉上永远叠加） |
| **是否可过渡动画<br />（支持transition)** | 不可                 | 不可                             | 可                     |
| **性能影响**                              | 重排+重绘            | 仅重绘                           | 仅重绘（可能合成层）   |

### 过渡动画的条件

CSS 过渡（`transition`）要求属性值在变化时 **能够被插值** ，即浏览器能计算出中间状态。

* **`opacity`** 是数值类型，0 → 1 有明确的中间值，所以支持平滑过渡。
* **`visibility`** 是离散属性（`hidden` / `visible`），无法插值。
* **`display`** 更是完全无法过渡，因为 `none` 与 `block` 等值之间没有数值上的连续过渡。

### 能否触发事件

元素能否触发鼠标/触摸等事件，取决于它 **在渲染树中是否占据布局区域** ，以及 **是否能够被命中测试** 。

* **`display: none`** 的元素不在渲染树中，没有布局框 → 不可能成为事件目标。
* **`visibility: hidden`** 的元素在渲染树中，占据空间，但大多数浏览器的实现会 **阻止它成为事件目标** （相当于 `pointer-events: none` 的效果）。
* **`opacity: 0`** 的元素完全在渲染树中，占据空间，且默认 **可以成为事件目标** （除非显式设置 `pointer-events: none`）。

可见，事件触发主要取决于 **元素是否在渲染树中且未被事件屏蔽。**
