# `css`样式的权重值

| 项目                                                                             | 权重   |
| -------------------------------------------------------------------------------- | ------ |
| ！important                                                                      | 正无穷 |
| 行间样式（style）                                                                | 1000   |
| id选择器(eg:#box)                                                                | 100    |
| class选择器(eg:.box)/属性选择器(eg:input[type="radio"])/伪类选择器（eg :hover ） | 10     |
| 标签选择器（div）/伪元素（eg ::before）                                          | 1      |
| 通配符选择器（*），组合器（+，>，～），否定伪类（:not）                          | 0      |

# 权重的“相加”方式

当你写一个选择器时，分别统计其中包含的各等级选择器数量，形成一个四元组 `(A, B, C, D)`。

 **比较规则** ：从左到右逐位比较，只要某一位数值更大，整个权重就更大，后面位不再比较。

## 示例

* `#nav .item a:hover`
  → ID 1个，类/伪类 2个（`.item` 和 `:hover`），元素 1个（`a`）
  → `(0, 1, 2, 1)`
* `#nav .item`
  → `(0, 1, 1, 0)`

比较时，先看 B 位：都是 1 → 看 C 位：前者 2 > 后者 1，所以 **第一个权重更高** ，即使后者元素选择器数量多也无关。

💥注意： `!important` 是另一个维度，它会覆盖普通声明的任何权重（除非多个 `!important` 之间再比较权重）。它不参与四元组的计算。

# 优先级的基本规则

1. `!important `优先级最高，但也会被权重高的 `!important`所覆盖
2. 不同权重，权重值高则生效；
3. 相同权重：后面出现的选择器生效；
4. 就近原则：如果在 `CSS`文件中定义的样式 `A`，在 `html`文件定义了 `B`，则应用 `B`；
5. `：not`否定伪类在优先级计算中不参加计算，但是，计算选择器数量时会把 `：not `当做普通选择器进行计数.

## 题目

1. 说出 `<p>`的颜色

   ```html
   <style>
           .b{
               color: blue;
           }
           .a{
               color: red;
           }
   </style>
   <body>
       <p class="a b">1232</p>
   </body>
   ```
   p为红色

   ```html
   <style>
           .a{
               color: blue;
           }
           .b{
               color: red;
           }   
       </style>
   </head>
   <body>
       <p class="b a">1232</p>
   </body>
   ```
   p依然为红色，因为在style中，.a和.b的权重相同，所以后面的会覆盖掉前面的。和在 `html`中使用 `a`和 `b`的顺序无关。
