# 块级格式化上下文(`Block Formatting Context`)

`BFC`是 `块级格式化上下文`，是一个**独立**的渲染区域，并且有一定的布局规则

# `BFC`创建方法

- **根元素（`<html>)`**
- **浮动**元素 ( `float` 不为 `none`)
- **绝对定位**元素( `position`: `absolute` / `fixed`)
- `overflow` 不为 `visible` 的块级元素
- `display:inline-block / table-cells / table-caption`
- `display` 的值为 `flow-root`的元素(`该元素会生成一个块级容器框，并且使用的是流布局。为里面内容创建新的块级格式化上下文。`)
- 多列容器（元素的 `column-count` 或 `column-width` 不为 `auto`）

# `BFC`范围

`BFC`包含创建该上下文元素的所有子元素，但不包括创建了新 `BFC`的子元素的内部元素。

```html
<div id='div_1' class='BFC'>
    <div id='div_2'>
        <div id='div_3'></div>
    </div>
    <div id='div_4' class='BFC'>
        <div id='div_5'></div>
    </div>
</div>
```

根据定义，`#div_1`创建了一个块格式上下文，这个上下文包括了 `#div_2、#div_3、#div_4`。由于 `#div_4`创建了新的 `BFC`，所以 `#div_5`就被排除在外层的 `BFC`之外。

同时，因为#div_1是容器，它不存在于自己创建的块级格式化上下文中，而是由父级格式化上下文进行定位。

只有根元素是特例，因为它没有父级的块级格式化上下文。

也就是说，**一个元素不能同时存在于两个 `BFC`中。**

这也是为什么 `BFC`可以解决浮动触发的问题，因为起到了隔离作用。

# 布局规则

#### 1.垂直方向：

`BFC`内部的盒子依次放置，相邻盒子，如果他们没有被非空内容，`padding`，`border`或 `clear`分隔开，就会发生 `margin`折叠

这里分两种情况

* **根元素（`<html>`）** ：它是初始 BFC 的根。根元素与它的直接子元素（通常是 `<body>`）之间的 margin  **会发生折叠** （这是规范特例，因为根元素没有父级 BFC）。
* **非根元素** ：如果一个元素 **创建了 BFC** （例如 `overflow: hidden`、`float: left`、`position: absolute` 等），那么该元素 **与它的子元素之间不会发生 margin 折叠** （BFC 隔离特性）。但如果一个非根元素 **没有创建 BFC** （普通块级元素），那么它与子元素之间 **仍然可能发生 margin 折叠** （例如父子折叠）。

#### 2.水平方向：

每个元素的左外边距，应该与包含块的左边缘相接触(对于从左往右的格式化，否则相反)。即使**存在浮动也是如此**

#### **3. `BFC`就是页面上的一个**隔离的独立容器，容器里面的子元素不会影响到外面的元素。（解决 `margin`塌陷）

#### 4. `BFC`的区域不会与浮动元素重叠

#### 5. 计算 `BFC`的高度时，**浮动元素也参与计算**(解决浮动元素父级高度塌陷)

### 应用

- 清除浮动（解决浮动元素父级高度塌陷)
- 分属于不同的 `BFC`时可以解决 `margin`塌陷
- 自适应两栏布局（1.元素的左外边距与包含块的左边缘相接触2.`BFC`的区域不会与浮动元素重叠3.块级元素的 `width`具有流体的特性，可以自适应撑满父级的宽度。）

  ```html
  <style>
  .left{
  	width:200px;
  	float:left;
  }
  .right{
  	overflow:hidden;
  }
  </style>
  <div class = 'left'></div>
  <div class = 'right'></div>
  ```

# BFC 与 IFC 的区别

| 对比维度               | BFC（块级格式化上下文）                                                                              | IFC（行内级格式化上下文）                                                                 |
| ---------------------- | ---------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------- |
| **产生条件**     | 根元素、浮动、绝对定位、`overflow: auto/hidden`、`display: inline-block/table-cell/flow-root` 等 | 块级容器内部**仅包含行内级元素** （无块级盒子）                                     |
| **内部元素类型** | 块级盒子（`block`、`flex`、`grid` 等）                                                         | 行内级盒子（`inline`、`inline-block`、匿名文本等）                                    |
| **排列方向**     | 垂直方向依次排列（从上到下）                                                                         | 水平方向依次排列（从左到右），超出宽度则换行形成多个行框                                  |
| **是否隔离**     | ✅**隔离容器** ：内部与外部互不影响（浮动不溢出、外边距不穿透、不与外部重叠）                  | ❌**不隔离** ：内部行框宽度受外部浮动影响，行框高度变化直接改变父容器高度           |
| **外边距折叠**   | ✅ 内部相邻块级盒子**会发生** margin 折叠（符合条件时）                                        | ❌**不会发生** margin 折叠（行内盒子上下 margin 无效，`inline-block` 不参与折叠） |
| **高度计算**     | 计算高度时**包含内部浮动元素**                                                                 | 高度由行框（各行的最高点/最低点）决定，不处理浮动（浮动会挤压行框宽度）                   |
| **垂直对齐**     | 块级盒子通过 `margin`、`padding` 或绝对定位调整位置                                              | 行内级盒子通过 `vertical-align` 在行框内对齐                                            |
| **典型应用**     | 清除浮动、防止外边距穿透、自适应两栏布局、封装独立组件                                               | 文本排版、水平居中（`text-align: center`）、垂直居中（借助 `vertical-align`）         |
